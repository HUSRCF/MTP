# Project Summary

## Goal
- Build a safe, evidence-gated MTP/premap path for vLLM/AWQ that can eventually prepare descriptor/address handles before demand.
- Current focus: move future WNA16 typed-slot materialization out of the hot WNA16 launch wrapper and into the producer/native-adapter package path, while keeping payload movement and current WNA16 kernel-arg replacement disabled except for explicit canaries.

## Decisions
- Do not pass typed tables as existing WNA16 args.
- Use an independent future WNA16 typed-slot ABI/variant path.
- Treat current typed-slot path as correctness / ABI evidence, not a TPOT performance win.
- Full payload transfer, ready credit, router mutation, descriptor-order execution, and current WNA16 arg reinterpretation remain out of scope unless explicitly enabled by a strict gate.
