## Open Issues
- Producer/native adapter still assigns rows from Python and copies staging buffers for most unique packages; TPOT remains far slower than production-like baseline.
- Table-content cache hit rate is low in the 8-sample smoke, so it is evidence that repeated table content is not common enough under the current table hash to fix the overhead.
- Need a lower-level native/C++ producer or an earlier table-construction contract that creates packed integer columns directly and avoids Python semantic row construction/copy for unique tables.
- Need decide whether to run a longer 32/128 strict gate after this refactor; 128 can take around an hour based on prior runs.
